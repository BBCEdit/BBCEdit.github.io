0.36.0

New Features

  • Added customisable 'places' to the file menu and file dialogue, increasing convenience, and solving the problem of the file dialogue not being able to access locations on different drives (a particular problem under Windows).
  • File dialogue defaults to directory of last loaded or saved program, rather than always defaulting to the location of BBCEdit.
  • Added HTML export.
  • Added option to open directory of currently loaded program with OS file manager.
  • Added option to open the directory where BBCEdit stores its config files with OS file manager.
  • 'Report Bug' menu option is changed to 'Debug BBCEdit', providing a place to list information about the current state of BBCEdit.
  • Moved 'recent file list' into its own menu, facilitating an increase in the number of recent files that are stored (doubling from 9 to 18).
  • Added menu item help on status-bar.
  • Bracket matching includes structure braces.

Bug Fixes

  • File dialogue does not crash when opening a root directory that does not contain any files or folders.
  • Accessing the menu-bar with Alt, works smoothly without locking up.
  • Insert line below (Ctrl+Return) works!
  • 'Strip Unused Line Numbers' does not crash (a typo had been introduced at some point).

Cosmetic

  • Menu-bar and tool-bar alignment is correct on both BBCSDL and BB4W.

0.35.0

New Features

  • Added option for limited single-line bracket matching.
  • Added option to preserve line indent as white space when using the Add REMs function.

Bug Fixes

  • IMPORTANT! Fixed tokenise routine (again) so that keywords that follow a string that contains a backslash are tokenised. This bug was possibly a regression caused by previously fixing a related bug.
  • When using 'Page Down' at the end of the program, the editor no longer overshoots the program.
  • The relative position of the currently edited line no longer jumps to the top of the editing pane when using Page-Up or Page-Down.
  • The main tool-bar updates (where previously it did not) when REMs are added, removed or inverted using the tool-bar.
  • Fixed the ANSI character edit function to correctly prevent characters outside of strings and comments being edited.
  • Fixed Ctrl+I/Tab to invert REMs on selected lines.

Structural

  • Significant changes to file and directory handling to improve compatibility across platforms, and to reduce (hopefully prevent) instances of files or directories that should be accessible, not being so from the load/save dialogues.
  • Significant code restructuring and some streamlining, leading to the removal of stringmod.bbc, arraymod.bbc, and dirmod.bbc.

Cosmetic

  • Changed character used to indicate that a menu item is 'ticked'.

0.34.2

New Features

  • Added ability to inspect the character values of the program line (text format not tokenised format) being edited. Select a single character in the normal way (perhaps with Shift+Right-Arrow) then use Ctrl-Left/Right-Arrow to traverse the edit line text. Pertinent information is displayed in the status bar. Use Ctrl-Page Up/Down to increment/decrement the value of the selected character.
  • Added option to purge broken links from the recent files list.
  • Added option to disable syntax colouring (might give a slight performance boost on slow systems).
  • Added warning when running BBCEdit under BB4W, when a program is loaded that has strings containing the control characters that are not part of the ANSI character set. BBCEdit currently has no work-around for dealing elegantly with these characters in the event that PRINT/VDU can't automatically print a dummy character in their place.

Bug Fixes

  • IMPORTANT! Fixed tokenise routine so that it does not corrupt line continuation comments that contain a period.
  • IMPORTANT! Fixed comments not being de-tokenised correctly, resulting in data corruption when comments contained character codes that corresponded with keyword tokens.
  • Fixed edit line changes being forgotten when opening the function list or making a 'Go To...' jump.
  • Fixed 'Go To...' dialogue not opening after a line selection.
  • Fixed variable names beginning with 'DEF' ending up in the function list.
  • Fixed line indentation not responding when certain changes were made, followed by cursor key up or down.
  • Fixed bug that prevented data recovery from working properly on next launch following a fatal error.
  • Fixed indentation problem exposed by Ceefax.bbc.
  • Fixed F1 key regression.

Cosmetic

  • Changed .BBC file icon to something more recognisable using the icon supplied with BBCSDL.
  • When running BBCEdit under BB4W and the DejaVuMono font is not installed, BBCEdit defaults to Courier New.
